# 添加开机自启动功能

## 项目现状分析

Bing Wallpaper Client 是一个基于 **Tauri v2** 的纯 Rust 系统托盘应用，功能为从 bing.wdbyte.com 获取每日壁纸并设置为桌面壁纸。

- 没有 Web 前端，所有交互通过系统托盘菜单完成
- 托盘菜单包含：壁纸刷新(中国/国际)、打开网站、退出
- 当前无开机自启动功能
- 支持 macOS 和 Windows（Linux 编译支持但未实现设壁纸）

## 方案：使用 tauri-plugin-autostart 官方插件

Tauri 官方提供了 [tauri-plugin-autostart](https://v2.tauri.app/plugin/autostart) 插件（当前版本 2.5.1），原生支持 **Windows、macOS、Linux** 三个平台：

- **macOS**: 通过 LaunchAgent 实现
- **Windows**: 通过注册表实现
- **Linux**: 通过 XDG autostart 实现

## 需要修改的文件

### 1. [src-tauri/Cargo.toml](src-tauri/Cargo.toml) — 添加依赖

在 `[dependencies]` 中添加：

```toml
tauri-plugin-autostart = "2"
```

### 2. [src-tauri/src/lib.rs](src-tauri/src/lib.rs) — 核心逻辑修改

**a) 注册插件**

在 `run()` 函数的 `tauri::Builder::default()` 链中注册 autostart 插件：

```rust
.plugin(tauri_plugin_autostart::init(
    tauri_plugin_autostart::MacosLauncher::LaunchAgent,
    None,
))
```

**b) 添加托盘菜单项**

在现有菜单中，分隔线之前添加"开机自启动"菜单项，带有勾选状态显示：

```rust
&MenuItem::with_id(app, "autostart", "开机自启动", true, None::<&str>)?,
```

需要在 `update_menu()` 函数中也加入此项，并根据当前自启动状态显示 `✓` 标记。

**c) 处理菜单点击事件**

在 `tray.on_menu_event` 的 match 分支中，新增 `"autostart"` 分支：

- 使用插件的 Rust API 检查当前状态 (`is_enabled`)
- 根据状态切换启用/禁用 (`enable` / `disable`)
- 更新菜单显示

**d) 初始状态加载**

在 `setup` 闭包中，初始化菜单时读取当前自启动状态，正确显示勾选标记。

### 3. [src-tauri/capabilities/default.json](src-tauri/capabilities/default.json) — 添加权限

虽然本项目仅使用 Rust API，但为规范起见，添加 autostart 权限：

```json
{
  "permissions": [
    "core:default",
    "opener:default",
    "autostart:allow-enable",
    "autostart:allow-disable",
    "autostart:allow-is-enabled"
  ]
}
```

## 菜单结构变更

变更前：

```
每日壁纸刷新(中国)
每日壁纸刷新(国际)
--------------
打开必应壁纸网站
退出
```

变更后：

```
每日壁纸刷新(中国)
每日壁纸刷新(国际)
--------------
开机自启动          (或 "开机自启动 ✓")
打开必应壁纸网站
退出
```

## 平台兼容性


| 平台      | 自启动机制         | 状态     |
| ------- | ------------- | ------ |
| macOS   | LaunchAgent   | 插件原生支持 |
| Windows | 注册表           | 插件原生支持 |
| Linux   | XDG autostart | 插件原生支持 |


以上三个平台的差异由 `tauri-plugin-autostart` 插件内部处理，无需额外编写平台特定代码。
