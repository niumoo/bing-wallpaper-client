# Bing Wallpaper Client 项目说明文档

## 项目概况

这是一个基于 **Tauri 2.0 + Rust** 的桌面系统托盘应用，没有可视窗口和前端 UI，所有逻辑集中在 Rust 后端。应用通过系统托盘菜单操作，自动从 [bing.wdbyte.com](https://bing.wdbyte.com) 获取每日必应壁纸并设置为桌面壁纸。

## 文档结构规划

`doc.md` 将包含以下章节：

### 1. 项目简介

- 应用名称: Bing Wallpaper、标识符: `com.wdbyte.bing.app`、版本: 0.1.0

### 2. 技术栈

- Tauri 2.0（桌面框架 + 托盘图标）
- Rust（核心逻辑）
- 关键依赖: `minreq`(HTTP)、`serde/serde_json`(JSON)、`uuid`(设备标识)、`winapi`(Windows 壁纸)、`open`(浏览器)

  - [src-tauri/src/main.rs](src-tauri/src/main.rs) - 程序入口

  - [.github/workflows/main.yml](.github/workflows/main.yml) - CI/CD

### 4. 核心功能详解

- **壁纸获取**: API 端点 `https://bing.wdbyte.com/zh-cn/today`（中国）和 `https://bing.wdbyte.com/today`（国际）
- **定时刷新**: 每 10 分钟（`REFRESH_INTERVAL = 600`），通过后台线程 + `AtomicBool` 实现取消
- **壁纸设置**: macOS 用 AppleScript (`osascript`)、Windows 用 `SystemParametersInfoA`
- **托盘菜单**: 中国/国际刷新模式切换（含勾选标记）、打开网站、退出
- **设备标识**: UUID 生成并持久化到 `device_uuid.txt`
- **缓存机制**: 已下载壁纸跳过重复下载

### 5. 数据流架构图

- 使用 mermaid 图展示: 托盘菜单 -> AppState -> API 请求 -> 下载 -> 设置壁纸 的完整流程

### 6. 状态管理

- `AppState` 结构体: `refresh_mode`（DailyChina/DailyGlobal/None）+ `timer_handle`
- 通过 `Mutex` 保护，由 Tauri `manage()` 管理

### 7. 数据存储

- macOS/Linux: `~/.bing-wallpaper-client/`
- Windows: `%APPDATA%/bing-wallpaper-client/`
- 存储: `device_uuid.txt` + 壁纸图片文件

### 8. 构建与运行

- 开发: `cargo install tauri-cli && cargo tauri dev`
- 构建: `cargo tauri build`
- Release 优化配置: LTO、strip、opt-level="s"

### 9. CI/CD

- GitHub Actions 自动构建: macOS ARM/Intel、Ubuntu、Windows
- 触发条件: `v`* 标签推送
- Windows 使用 WiX 打包安装程序

### 10. 平台支持说明

- macOS: 完整支持（AppleScript 设置壁纸 + Accessory 模式隐藏 Dock 图标）
- Windows: 完整支持（WinAPI 设置壁纸）
- Linux: 构建支持，但壁纸设置功能未实现
